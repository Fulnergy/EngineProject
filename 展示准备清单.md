### 2. 建议演示顺序

**第一段：DDL + 基础CRUD**
```sql
create table t( id int, name char, age int, gpa float);
show tables;
describe t;
insert into t (id, name, age, gpa) values (1, 'a', 18, 3.6);
select * from t;
update t set t.name = 'apple' where t.id = 1;
delete from t where t.id = 1;
```

**第二段：查询**
```sql
select t.id, t.name from t where t.age > 18;       -- 投影+范围
select * from t where t.age = 19 and t.gpa > 3.5;  -- AND
select * from t where t.age = 18 or t.age = 20;    -- OR
select count(*) from t;                              -- COUNT
select count(*) from t where t.age > 18;             -- COUNT+条件
```

**第三段：EXPLAIN 看执行计划**
```sql
explain select t.id, t.name from t where t.age > 18;
```

**第四段：索引**
```sql
create index idx_age on t(age);
select * from t where t.age = 19;   -- IndexScan
drop index idx_age;
```

**第五段：事务**
```sql
begin;
insert into users (id) values (1);
savepoint sp1;
insert into users (id) values (2);
rollback to savepoint sp1;   -- 回滚到sp1
insert into users (id) values (3);
commit;
select * from users;          -- 验证只有1和3
```

**第六段：DROP TABLE**
```sql
drop table users;
show tables;
```

## 四、QA 准备 — 高频问题

### 架构

**Q1: SQL从输入到输出经历了什么？**
```
SQL字符串 → JSqlParser(语法树) → LogicalPlanner(逻辑算子树)
→ PhysicalPlanner(物理算子树) → 火山模型(Begin/Next/Current/HasNext/Close)
→ BufferPool(页缓存) → DiskManager(磁盘读写) → 打印
```

**Q2: 逻辑算子 vs 物理算子的区别？为什么分两层？**
- 逻辑算子：描述"做什么"（扫描、过滤、投影），与实现无关
- 物理算子：描述"怎么做"（SeqScan还是IndexScan）
- 分两层为了优化：同一逻辑可以选不同物理实现，上层不用改

**Q3: 火山模型是什么？**
每个算子实现 Begin/Next/Current/HasNext/Close 五个方法。数据从底层向上"拉取"（pull模型），像岩浆从火山底往上喷发。好处：懒执行、一次一行、可组合。

### 存储

**Q4: LRU和Clock的区别？**
- LRU：精确淘汰最久未使用。访问O(n)移动链表，淘汰O(1)
- Clock：近似LRU。访问O(1)设refBit，淘汰O(n)转圈。真实数据库倾向Clock

**Q5: Pin/Unpin的作用？**
- Pin：标记frame正在使用，不可淘汰
- Unpin：释放保护，可淘汰。pin_count=0时Unpin加入Replacer

**Q6: 脏页是什么？**
内存中修改过但未写回磁盘的页。淘汰脏页前必须FlushPage，否则丢数据。

**Q7: 为什么一次读一页(4KB)而不是一行？**
磁盘随机访问毫秒级，顺序读很快。一次读一页摊销量I/O开销。每页包含多个record slot + bitmap。

### 索引

**Q8: B+树结构？内部vs叶子节点？**
- 内部节点：只存key+子指针，用于路由
- 叶子节点：存key+RID+next链表指针
- 所有数据在叶子层，任何查询都走到叶子

**Q9: B+树 vs B树？**
- B+树：数据只在叶子，叶子有链表，范围查询O(log N + K)
- B树：数据在所有节点，范围需中序遍历。数据库都用B+

**Q10: B+树分裂过程？**
1. 叶子key数 > maxKeys → 拆两半
2. 右半第一个key提升到父节点
3. 父节点溢出 → 递归分裂
4. 根分裂 → 树长高（树长高的唯一方式）

**Q11: CREATE INDEX做了什么？**
1. 更新MetaManager元数据
2. 创建空B+树
3. SeqScan全表扫描，逐行插入B+树
4. 存入DBManager.indexes

**Q12: IndexScan vs SeqScan？**
- IndexScan：从B+树取RID，按page排序后直接GetRecord
- SeqScan：逐页逐slot扫描bitmap

### 事务

**Q13: 事务怎么实现的？**
- BEGIN：复制数据库目录到临时快照
- COMMIT：持久化+删快照
- ROLLBACK：清空目录，快照复制回来
- 基于文件系统，简单但足够本项目

**Q14: SAVEPOINT栈语义？**
- 同名保存点用Deque push/pop
- 后创建的遮蔽先创建的
- ROLLBACK TO / RELEASE 操作栈顶

**Q15: ROLLBACK vs ROLLBACK TO SAVEPOINT？**
- ROLLBACK：回滚整个事务到BEGIN前
- ROLLBACK TO：只回滚到指定保存点

**Q16: ACID实现了哪些？**
- Atomicity：ROLLBACK回滚
- Durability：COMMIT写磁盘
- Consistency：快照保证
- Isolation：单用户模式，未实现

public int Victim() {
    if (LRUList.isEmpty()) return -1;

    LRUHash.remove(v);
    return v;
}
// Clock: 转圈找refBit=0
public int Victim() {
    while (checked < capacity * 2) {

**Q18: FilterOperator的火山模型？**
```java
public boolean hasNext() {
    while (child.hasNext()) {
        child.Next();
        if (child.Current()满足条件) return true;
    }
    return false;
}
```
